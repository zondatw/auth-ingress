# Contract: Visual System

The refreshed UI must present auth-ingress as a coherent technical control
surface across all visible pages.

## Required page coverage

- Sign-in
- First-install setup required
- Change password
- Reset password
- Portal service list
- Access denied/error page
- Admin users list and detail
- Admin groups list and detail
- Admin services list/edit
- Admin audit events

## Required component coverage

- Product shell and navigation
- Page title/header blocks
- Panels and elevated cards
- Summary cards and relationship/dependency cards
- Forms, labels, inputs, selects, checkboxes, and field errors
- Primary, secondary, disabled, and destructive actions
- Tables and dense data regions
- Status chips and state badges
- Alerts, notices, confirmations, and empty states
- Secret/temporary-password display and copy feedback

## Visual expectations

- The default impression is dark, technical, and focused.
- Page hierarchy is immediately clear: product shell, section, primary title,
  summary/context, then detailed controls.
- Data-dense regions remain readable through spacing, grouping, and contrast.
- Long identifiers, URLs, emails, slugs, and temporary passwords wrap safely.
- Mobile layouts keep navigation and primary actions reachable.

## Non-goals

- Do not copy unrelated content from the reference project.
- Do not introduce externally hosted design assets.
- Do not change business workflows or authorization rules.
